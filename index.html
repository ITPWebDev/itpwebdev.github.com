#1 HTML & CSS

HTML

HTML, HyperText Markup Language, is exactly that, a markup language for text on internet. A markup language is a computer language to make text and media readable by a computer.

A word processor like Pages or Word is doing its own markup in the background to give you nice paragraphs and fonts and colors. Copying and pasting from a word processor does not retain this markup and anyway Pages and Word use proprietary markup, so you can't have it.

HTML is made up of text and media enclosed in elements forming a tree structure, which is mimicked in text through the convention of tabbing.

<html>
  <head>
    <title>HELLO WORLD</title>
  </head>
  <body>
    <div>
      <h1>I'm hungry.</h1>
      <p>So give me some food.</p>
    </div>
  </body>
</html>

Elements need to be closed with a </p>. Elements can be nested, enclosed within other elements.

Marking Up a Document

To better explain HTML we'll work through an example of converting a PDF to an HTML document. We'll use the syllabus for the Cabinets Of Wonder class taught at ITP.

If we just copy and paste the text into a .html file we get one big chunk of text. What happened to all of the paragraphs? If you view the source code of the page all of the paragraphs are still there but are being ignored by the browser.

We have an html document but there is no HTML in it. So how do we add it? First we need to address the structure of HTML.

<html></html>

Everything in an html document must be inside of these tags. These tags tell the browser that you want an html document and not an xml, json, javascript, css or any other sort of document.

<head></head>

The head does not display on the page, it is extra information and includes references to stylesheets and javascript files. The only mandatory element inside of head is <title></title>.

<body></body>

The body is where everything to be displayed in the browser is contained. At this point the structure of our document looks like this:

  html
    -head
      -title
    -body

Our example now has the html, head, title, and body elements. But it looks the same.

Our document has distinct sections, the class info, background, overview, and schedule. We'll start by splitting up the file into its parts. The <div> element will be used here. If we revisit the example we see that each div causes a new line. We're getting close.

Behind the scenes the div elements are breaking up the text into separate boxes or block. HTML elements are either block or inline. A block element puts everything inside into a box. Inline elements do not break the flow of the page. This might sound weird now but just bear with us, it'll make sense in time.
